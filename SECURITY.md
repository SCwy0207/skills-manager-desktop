# Security Policy

## Supported versions

Security fixes are provided for the latest published release. Pre-release builds and locally modified binaries are not supported.

| Version | Supported |
| --- | --- |
| 1.0.x | Yes |
| Earlier internal builds | No |

## Reporting a vulnerability

Please do not disclose security issues in a public issue, discussion, screenshot or log attachment.

Use the repository's **Security → Advisories → Report a vulnerability** flow. Include the affected version, operating system, impact, minimal reproduction and any suggested mitigation. Remove API keys, session text, Skill content, usernames and absolute local paths before attaching diagnostics.

The maintainer should acknowledge a report within seven days, provide an initial assessment within fourteen days and coordinate disclosure after a fix is available. These are response targets rather than a warranty.

If private vulnerability reporting is not visible, the repository owner must enable it under **Settings → Security → Private vulnerability reporting** before the public launch.

## 安全漏洞报告

请勿在公开 Issue、Discussion、截图或日志中披露安全漏洞。请使用仓库的 **Security → Advisories → Report a vulnerability** 私密报告入口，并在附件中移除 API Key、会话正文、Skill 内容、用户名和本机绝对路径。
